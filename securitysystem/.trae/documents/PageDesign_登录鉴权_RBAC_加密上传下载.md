# 页面设计说明（桌面端优先）

## 全局设计（适用于全部页面）
### Layout
- 桌面端：页面最大宽度 1200px，主体内容居中；顶部固定导航（workbench/admin）或纯内容（auth）。
- 布局：导航/表单区采用 Flexbox；表格与卡片区采用 CSS Grid（12 栅格概念）。
- 响应式：>=1024px 为主；768-1023px 缩为单列；<768px 表格转为卡片列表。

### Meta Information
- Title 模板：{页面名} - QAR 安全系统
- Description：支持登录鉴权、加密上传、按权限解密下载与全量导出。
- Open Graph：og:title/og:description/og:type=website

### Global Styles（Design Tokens）
- 背景：#0B1220（深色）或 #F6F7FB（浅色，默认推荐浅色）
- 主色：#2563EB（按钮/链接）
- 危险色：#DC2626（删除/降权等高风险操作）
- 成功色：#16A34A（上传成功、下载成功提示）
- 字体：系统字体；字号 12/14/16/20/24
- Button：Primary（主色填充）、Secondary（描边）、Danger（危险色）；hover 提升亮度 5-8%
- Link：主色 + underline on hover
- Toast/Alert：顶部右侧轻提示；表单错误在输入框下方

## 1) 登录/注册页（/auth）
### Page Structure
- 单页卡片布局：居中卡片（宽 420-480px），上方 Logo/标题，下方 Tab：登录/注册。

### Sections & Components
1. 顶部区
   - Logo（文字版即可）
   - 标题：欢迎使用 QAR 安全系统
   - 副标题：登录后可加密上传并按权限解密下载
2. 登录 Tab
   - 输入：邮箱/用户名、密码
   - 按钮：登录（Primary）
   - 辅助：登录中 loading；错误提示（账号不存在/密码错误/服务异常）
3. 注册 Tab
   - 输入：邮箱/用户名、密码、确认密码
   - 按钮：注册（Primary）
   - 规则提示：密码强度与必填校验（最小可用即可）
4. 会话提示
   - 已登录状态：显示“已登录，将跳转工作台”并提供“前往工作台”按钮

Interaction States
- 表单校验：必填、密码一致；提交后按钮禁用避免重复提交。

## 2) 数据工作台（/workbench）
### Page Structure
- 顶部导航 + 两列布局（桌面端）：左侧“上传区”，右侧“我的数据列表”。
- 右侧列表支持选中一条记录，在列表行内提供“下载”操作。

### Sections & Components
1. Top Nav
   - 左：产品名
   - 中：导航（工作台；若角色为 admin 则显示“管理后台”入口）
   - 右：当前用户信息（role 标签）+ 退出登录
2. 加密上传区（左列 Card）
   - 上传方式：文件上传（input[type=file]）为主；可选文本粘贴区（若你现有项目以 JSON 文本为主）
   - 策略输入：policy（默认可填 role:admin 或按你现有接口约定）
   - 按钮：开始上传并加密
   - 结果回显：上传成功提示；展示密文长度/策略/记录 ID（不直接展示大段密文）
3. 我的数据列表（右列 Table）
   - 列：文件名、大小、上传时间、策略、操作
   - 操作：下载（触发解密下载）
   - 空状态：无数据时提示“先上传一份数据”
4. 下载提示（Modal/Drawer）
   - 展示：下载将进行服务端解密；无权限/解密失败的错误说明

Interaction States
- 上传中：进度条（可选）+ 禁用按钮
- 下载中：行内 spinner
- 鉴权失效：统一拦截提示“会话已过期，请重新登录”

## 3) 管理后台（/admin，仅 Admin）
### Page Structure
- 顶部导航 + 左侧二级菜单 + 右侧内容区。
- 二级菜单：用户与权限 / 全量数据管理。

### Sections & Components
1. 访问控制
   - 非 admin 访问时：展示无权限页面（403 文案）并提供返回工作台按钮
2. 用户与权限（表格）
   - 列：用户标识、当前角色、创建时间、操作
   - 操作：切换角色（user/admin）
   - 风险确认：降权/升权二次确认（Confirm Dialog）
3. 全量数据管理（表格）
   - 列：记录 ID、归属用户、文件名、时间、大小、策略、操作
   - 操作：单条下载（按需求可下载解密内容）
4. 全量下载
   - 主按钮：导出全部数据（Primary）
   - 说明：导出可能耗时；执行中显示 loading；成功后自动下载文件

Interaction States
- 高风险操作（角色变更、全量导出）均需二次确认
- 导出任务进行中：顶部提示“正在导出，请稍候”
